# KnowGate Helm Chart

Production Helm chart for the KnowGate platform (FastAPI + Celery + Next.js + LiteLLM). Targets Kubernetes 1.28+.

## TL;DR

```bash
helm repo add knowgate https://knowgate.example.com/charts
helm install knowgate knowgate/knowgate \
  --namespace knowgate \
  --create-namespace \
  --values values.prod.yaml
```

## Prerequisites

- Kubernetes cluster 1.28+
- Helm 3.10+
- Prometheus Operator (if `serviceMonitor.enabled=true`)
- cert-manager (if using TLS via Ingress annotations)
- Sealed Secrets controller (if using `sealed-secrets.yaml`)

## Chart contents

- Deployments: `api`, `worker`, `beat`, `litellm`
- Services: `api` (ClusterIP:8000), `litellm` (ClusterIP:4000)
- HorizontalPodAutoscalers: `api`, `worker`
- PodDisruptionBudgets: `api`, `worker`
- Ingress (nginx or traefik)
- NetworkPolicy
- ServiceMonitor
- Backup CronJob (pg_dump + qdrant snapshot)

External dependencies (PostgreSQL, Redis, Qdrant, MinIO) are NOT bundled by default; set the corresponding `*.host` and `existingSecret` values to point at managed services or the bundled subcharts.

## Custom values example

`values.prod.yaml`:

```yaml
global:
  env: production
  domain: knowgate.acme.com

api:
  replicas: 5
  hpa:
    minReplicas: 5
    maxReplicas: 20

worker:
  hpa:
    maxReplicas: 10

ingress:
  className: nginx
  annotations:
    cert-manager.io/cluster-issuer: letsencrypt-prod
  hosts:
    - host: knowgate.acme.com
      paths:
        - path: /
          pathType: Prefix
  tls:
    - secretName: knowgate-tls
      hosts:
        - knowgate.acme.com

postgres:
  host: postgres.knowgate-data.svc
  existingSecret: knowgate-db-credentials
  usernameKey: username
  passwordKey: password

redis:
  host: redis.knowgate-data.svc
  existingSecret: knowgate-redis
  passwordKey: password

secrets:
  existingSecret: knowgate-secrets
```

## Install / upgrade / uninstall

```bash
# Install
helm install knowgate ./deploy/helm/knowgate \
  --namespace knowgate --create-namespace \
  --values values.prod.yaml

# Upgrade
helm upgrade knowgate ./deploy/helm/knowgate \
  --namespace knowgate \
  --values values.prod.yaml

# Uninstall (PVCs are NOT removed automatically)
helm uninstall knowgate --namespace knowgate
```

## Secrets

The chart does NOT create any Secret. It references a Secret you provision out-of-band:

1. Use Sealed Secrets (see `templates/sealed-secrets.example.yaml`)
2. Or use an external operator (External Secrets Operator, HashiCorp Vault, etc.)

Required keys in `knowgate-secrets`:
- `OPENAI_API_KEY`
- `DB_PASSWORD` (and `DB_USER`, `DB_HOST`, `DB_NAME`, `DB_PORT` if not in values)
- `REDIS_PASSWORD`
- `MINIO_ROOT_USER`, `MINIO_ROOT_PASSWORD`
- `JWT_PRIVATE_KEY`, `JWT_PUBLIC_KEY`
- `ENCRYPTION_KEY`
- `GOOGLE_OAUTH_CLIENT_ID`, `GOOGLE_OAUTH_CLIENT_SECRET`
- `NOTION_API_KEY`
- `SMTP_PASSWORD`

## Backup

The bundled `CronJob` runs daily at 02:00 UTC by default. It:
1. Dumps PostgreSQL to local disk
2. Uploads the dump to S3
3. Triggers a Qdrant collection snapshot and uploads it
4. Prunes local files older than `backup.retentionDays`

Restore procedure is documented in `docs/know-gate/runbook-restore.md` (TODO).

## Network policy

By default, the chart creates a `NetworkPolicy` that:
- Allows ingress on port 8000 from any namespace (fronted by ingress controller)
- Allows ingress on port 8000/9090 from the `monitoring` namespace (Prometheus scrape)
- Allows egress to in-cluster Postgres, Redis, Qdrant, MinIO, LiteLLM
- Allows DNS egress
- Allows egress to OpenAI/Anthropic (toggleable)

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `PVC Pending` | StorageClass `gp3` not available | Set `*.storageClass` to your cluster's default. |
| `ImagePullBackOff` on api/worker | Wrong image tag or missing pull secret | Set `api.image.tag` or `global.imagePullSecrets`. |
| Pod `OOMKilled` | Resource limits too tight | Increase `api.resources.limits.memory`. |
| Backup pod fails with `connection refused` | Wrong `postgres.host` | Verify `values.yaml` and the DB Service. |
| ServiceMonitor shows `0 active targets` | Prometheus not in `monitoring` namespace | Adjust `serviceMonitor.prometheusNamespace`. |
| `celery inspect ping` fails in worker/beat liveness | Worker too slow to boot | Increase `worker.livenessProbe.initialDelaySeconds`. |
